## 3. 行動特性フレームワーク

### 選定: DISC（カスタム行動スタイル診断）

DISC / MBTI / PCM を比較検討し、以下の理由でDISCを採用:

| 評価項目 | DISC | MBTI | PCM |
|---------|------|------|-----|
| 3問での簡易判定 | **可能**（精度は中程度） | 不可能 | 不可能 |
| ライセンス | **パブリックドメイン** | 要ライセンス | 要ライセンス+認定 |
| ビジネスネットワーキング適性 | **最適** | 中 | 高だが重厚 |

**注意**: 「DiSC(R)」はWiley社の商標。本システムでは「行動スタイル診断」「四元素タイプ」等の独自名称を使用し、DISCとは明言しない。

### 四元素タイプ

| DISC | 和名 | EN | 色コード | バッジ | コンセプト |
|------|------|----|---------|----|-----------|
| D（主導型） | **炎** | Fire | #CF2030 | 🔥 | 情熱・決断・推進力。炎のように周囲を動かす |
| I（感化型） | **風** | Wind | #2196F3 | 🌀 | 社交・巻き込み・拡散力。風のように人をつなぐ |
| S（安定型） | **大地** | Earth | #FFC107 | 🌿 | 安定・信頼・継続力。大地のように支える |
| C（慎重型） | **水** | Water | #4CAF50 | 💧 | 分析・正確・適応力。水のように深く流れる |

### DISCサブタイプ（12パターン）

メインタイプ + サブタイプの組み合わせで12パターンに細分化:

| # | サブタイプ | 和名 | 特徴 |
|---|----------|------|------|
| 1 | D（純粋D） | 炎 | 結果重視、直接的 |
| 2 | Di | 炎風 | 行動的で社交的 |
| 3 | iD | 風炎 | エネルギッシュ |
| 4 | i（純粋I） | 風 | 楽観的、協力的 |
| 5 | iS | 風大地 | 温かく支持的 |
| 6 | Si | 大地風 | 協調的で穏やか |
| 7 | S（純粋S） | 大地 | 安定的、忍耐強い |
| 8 | SC | 大地水 | 慎重で着実 |
| 9 | CS | 水大地 | 正確で協調的 |
| 10 | C（純粋C） | 水 | 分析的、品質重視 |
| 11 | CD | 水炎 | 論理的で挑戦的 |
| 12 | DC | 炎水 | 決断力と分析力 |

### BNI集団での想定分布

一般人口ではS型が最多（32-69%）だが、BNIメンバーはネットワーキング組織の特性上I型・D型に偏る傾向がある。

**対策**: マッチングアルゴリズムでS/C型を各グループに最低2名ずつ均等配置。

#### F-004: 行動タイプ判定（新規）

**スコアリングロジック**:
```
Q7-Q9の各回答を集計:
  A選択 → D(炎)スコア +1
  B選択 → I(風)スコア +1
  C選択 → S(大地)スコア +1
  D選択 → C(水)スコア +1

メインタイプ = 最高スコアのタイプ
サブタイプ = 2番目に高いスコアのタイプ
（同点の場合: Q7 > Q8 > Q9 の優先度で先の質問の回答を優先）

サブタイプ判定:
  メインとサブのスコア差が2以上 → 純粋タイプ（例: D=3, I=0 → 純粋D「炎」）
  メインとサブのスコア差が1以下 → 複合タイプ（例: D=2, I=1 → Di「炎風」）
```

**出力**: `disc_main`（D/I/S/C）、`disc_sub`（D/I/S/C）、`disc_label`（「炎」「炎風」等）

#### F-006: AIグループマッチング（変更）

**マッチングルール（プロンプトに組み込み、優先度順）**:

1. **パワーチーム編成（最重要）**:
   - 同じtarget_customersを持つ異業種同士を同グループに（同顧客層マッチ）
   - categoryの意味的な補完関係をGeminiが自然言語で判断（例: 不動産×リフォーム×保険×税理士の紹介チェーン）
   - **「このグループならリファーラルが生まれる」**組み合わせを最優先

2. **同業種回避**:
   - 同じindustry_groupの人は同グループに3-4名まで（グループ20名中）
   - 完全回避が無理な場合のフォールバック値

3. **行動タイプの多様性**:
   - 各グループに炎(D)/風(I)/大地(S)/水(C)の全4タイプを含める
   - S型・C型はBNI集団で少数のため、各グループに最低2名ずつ保証

4. **チャプター分散**:
   - 同チャプターのメンバーは同グループに3名まで

5. **グループサイズ均等化**:
   - 10グループ × 約20名、差は最大±3名
   - 各グループ内の2テーブルも均等（差±1名）

**API設定**:
- モデル: `gemini-2.0-flash`（Gemini 3.1 Flash）
- temperature: 0.2
- responseMimeType: `application/json`
- maxOutputTokens: **16384**（v1の8192から増加。200名10グループの詳細出力に対応）

#### F-007: バリデーション・調整（変更）

**チェック項目**:
1. 全回答済み参加者がいずれかのグループに割り当てられているか（漏れチェック）
2. 各グループに4つの行動タイプが全て含まれているか（多様性チェック）
3. 各グループのS/C型が最低2名いるか（少数タイプ保証チェック）
4. 同グループに同じindustry_groupが4名以上いないか（同業種チェック）
5. 同グループに同じchapterが3名以上いないか（チャプターチェック）
6. 各グループが17-23名の範囲内か（サイズチェック）
7. 各グループ内の2テーブルが均等か（テーブルバランスチェック）
8. 参加者IDの重複がないか

**自動修正**: 違反があれば隣接グループとメンバーをスワップして修正（行動タイプバランスを維持しつつ）
**リトライ**: 自動修正で解決できない場合、Gemini APIを再呼び出し（最大3回）

#### F-010: スクリーン発表モード（変更）

- **Phase 1（メンバー発表）**: グループ番号（大きく）+ メンバー一覧（名前+チャプター+業種のみ）。グループ名は「???」で非表示
- **Phase 2（Reveal!）**: 「Reveal!」ボタン（またはキーボード`R`キー）で公開:
  - グループ名（パワーチームテーマ）が出現
  - 行動タイプバッジ + 構成バー（炎:5 風:6 大地:5 水:4）
  - シナジー理由（パワーチーム × 行動特性の複合説明）
  - メンバーアバターが行動タイプ色に変化
- **演出**: グループが1つずつ順に表示。5秒間隔で自動送り or クリック手動送り
- **デザイン**: ダーク背景 + BNIブランドカラー + 四元素カラー（炎赤/風青/大地黄/水緑）
- **進捗**: "Group 1/11" 表示（10グループ + セレンディピティ）
- **セレンディピティ**: 最後（11番目）に金色ボーダーの特別デザインで表示

#### F-011: 名前/電話番号検索モード（変更）

- **入力**: 名前 or 電話番号（自動判定。数字3桁以上なら電話番号検索、それ以外は名前検索。部分一致）
- **出力**: グループ番号 + グループ名 + 行動タイプバッジ + テーブル番号 + 同グループメンバー一覧（行動タイプ + 業種付き）+ シナジー理由
- **ハイライト**: 検索した本人を金色ハイライトで強調
- **多言語**: プレースホルダ・ラベルを日英バイリンガル
- **実装方式**: 結果JSONを埋め込んだ静的HTML（v1と同様）

#### F-014: QRランディングページ（新規）

- **概要**: QRコード読取後に表示される振り分けページ
- **選択肢**:
  1. 「まだ回答していない方」→ Googleフォームへ遷移
  2. 「回答済みの方」→ 名前/電話番号入力でグループ検索画面へ
- **実装方式**: 静的HTML（GitHub Pages）

## 7. 技術選定

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|
| フォーム | Google Forms | スプレッドシート連携、バイリンガル対応可 |
| データストア | Google Sheets | フォーム自動保存、行動タイプスコア算出、結果管理 |
| バックエンド | Google Apps Script | Sheets/Gemini APIとネイティブ連携 |
| AI | Gemini 3.1 Flash (gemini-2.0-flash) | 低コスト、JSON mode、大規模入力対応 |
| フロントエンド | 静的HTML (JSON埋め込み) | 同時アクセス問題回避、GitHub Pages |
| 配信 | WhatsApp + QRコード | 全参加者到達の確実性 |

**GASファイル構成（4ファイル）**:
- `Config.gs` — グループ数、行動タイプ設定、API Key等
- `DataTransform.gs` — スプレッドシート→JSON変換 + 行動タイプ判定 + 重複チェック
- `GeminiMatching.gs` — Gemini API呼び出し + グループベース出力パース
- `TableOptimizer.gs` — バリデーション + 自動修正 + セレンディピティ割当

## 10. リスクと対策

## 12. 用語集

| 用語 | 定義 |
|------|------|
| 行動スタイル診断 | 人の行動パターンを4タイプに分類する簡易診断。DISC理論（1928年Marston博士提唱、パブリックドメイン）をベースにした独自版 |
| 四元素タイプ | 本システム独自の行動タイプ呼称。炎(D)/風(I)/大地(S)/水(C) |
| サブタイプ | メイン+サブの組み合わせで12パターン（Di, iD, iS等） |
| パワーチーム | 業種の組み合わせでリファーラル（ビジネス紹介）が生まれるチーム。マッチングの主軸 |
| セレンディピティグループ | アンケート未回答者用の特別グループ。「予期せぬ出会い」の意 |
| Reveal演出 | メンバー発表→ネットワーキング後に「実はこのグループは○○のパワーチームでした」とシナジー理由+行動特性を公開するゲーム演出 |
| コンタクトサークル | BNIにおける、互いのクライアントを紹介し合える異業種メンバーのグループ |
| リファーラル | BNI用語で「ビジネス紹介」 |
| シナジー | 2名以上が組むことで生まれるビジネス効果 |
| チャプター | BNIの地域グループ |
